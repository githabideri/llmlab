# Qwen3.5-35B-A3B (Q4_K_M) — Preliminary llmlab Report (2026-02-26)

- **Model (GGUF):** `Qwen_Qwen3.5-35B-A3B-Q4_K_M.gguf`
- **Base model:** https://huggingface.co/Qwen/Qwen3.5-35B-A3B

- **Backend:** llama.cpp
- **Hardware:** 2× RTX 3060 12GB

## Summary
This is a **preliminary** stability report from llmlab interactive testing. The run showed persistent repeated tool-call behavior (primarily repeated web searches) without converging to a useful final answer.

## Observed behavior
- User asked for available tools and weather-tool capability.
- Model repeatedly emitted tool calls (`web_search`) in long chains.
- Answer quality degraded because generation budget was consumed by repeated tool invocations.
- Context depth increased continuously until fallback flow failed on context limits.

## Performance snapshot (latest loop-heavy block)
- Requests in block: **122**
- Context depth: **29,752 → 119,230**
- Mean TG speed: **30.87 tok/s**
- TG speed trend: **38.99 → 23.81 tok/s**
- Mean generated tokens/request: **~51** (mostly tool-call payload)

## Preliminary interpretation
- The setup is currently not stable for normal llmlab interactive use with this model.
- Main risk is runaway tool-call loops causing context inflation and eventual failure.
- Further root-cause investigation is required before re-enabling this model in room-default operation.

## Operational decision (temporary)
- Move llmlab default operation to **GLM-4.7-Flash** for now.
- Keep Qwen3.5 for isolated experiments and controlled repro runs.

## Next investigation tasks
1. Reproduce with minimal tool set and fixed prompt template.
2. Compare streaming vs non-streaming runtime path behavior end-to-end.
3. Add hard loop guardrail policy for repeated identical tool calls.
4. Re-test after upstream fixes and re-evaluate.
