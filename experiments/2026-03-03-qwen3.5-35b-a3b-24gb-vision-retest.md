# Qwen3.5-35B-A3B Q4_K_M — 24GB Vision Retest (2026-03-03)

## Goal
Re-check whether Qwen3.5-35B-A3B can run as a single text+tools+vision model on dual RTX 3060 (24GB total), after earlier vision OOM reports.

## Setup
- **Hardware:** 2× RTX 3060 12GB (24GB total)
- **Runtime:** llama.cpp `llama-server`
- **Model:** `Qwen_Qwen3.5-35B-A3B-Q4_K_M.gguf`
- **Projector:** `mmproj-Qwen_Qwen3.5-35B-A3B-f16.gguf`
- **Profile:**
  - `--ctx-size 98304`
  - `--split-mode layer`
  - `--gpu-layers 99`
  - `--parallel 1`
  - `--cache-type-k q8_0 --cache-type-v q4_0`
  - `--no-mmproj-offload`
  - `--flash-attn on`

## Commands
```bash
# Verify runtime process and flags
ps -eo pid,args | grep '[l]lama-server'

# Verify loaded artifacts
ls -lh /mnt/models/gguf/qwen3.5-35b-a3b/*.gguf

# Runtime VRAM
nvidia-smi --query-gpu=name,memory.total,memory.used --format=csv,noheader

# Vision preprocessing timings
grep 'process_chun: image processed' /var/log/llama-server.log | tail
```

## Observations

### Fitment
- Service stayed up with model + mmproj loaded.
- VRAM remained below per-card limits during active use:
  - GPU0: ~11.9/12.3 GB
  - GPU1: ~11.3/12.3 GB

### Vision path
- Image understanding responses returned successfully.
- Preprocessing latency scaled steeply with resolution:
  - ~128px class: ~0.3s
  - ~512px class: ~5.3s
  - ~1024px class: ~32s

### Text/tool behavior
- Short prompt checks completed without immediate repeated-tool-call spirals.
- Historical loop risk remains from earlier long sessions and should still be treated as unresolved for unattended use.

## Metrics snapshot
From timing lines in recent serving logs at ~32k prompt depth:
- Prompt eval: ~725–984 tok/s (depends on checkpoint reuse)
- Generation eval: ~44–45 tok/s

## Conclusion
Qwen3.5-35B-A3B **does fit on 24GB with vision** when the runtime profile is constrained (96k ctx, layer split, single slot, no mmproj offload). 

This invalidates the blanket “24GB can’t do vision” conclusion. The better statement is:
- **default/heavier profile:** can OOM,
- **tuned profile:** works.
