# Nanbeige4.1-3B Quant Sweep (24GB, Long Context)

**Date:** 2026-02-23  
**Model:** Nanbeige4.1-3B (GGUF)  
**Quants tested:** `i1-IQ4_NL` (2.2G), `i1-Q6_K` (3.1G), `Q8_0` (3.9G)  
**Hardware:** 2× RTX 3060 12GB (24GB total)  
**Runtime:** llama.cpp server, `-sm layer -fa 1 -ctk q8_0 -ctv q4_0`

## Goal

Measure prompt processing (PP) + generation (TG) behavior at high and very high context, then compare quant trade-offs.

## Method

- Endpoint: OpenAI-compatible `/v1/completions`
- Prompt: repeated token stream to target context depth
- `cache_prompt=false`, `temperature=0`, `max_tokens=128`
- Two sweeps:
  1. 100k / 120k / 128k
  2. 192k / 261k (near 262,144 train context)

## Findings

1. **At 100k-ish context**, Q6/Q8 are better at PP ingest, but IQ4_NL still leads TG.
2. **At deep context (192k+)**, IQ4_NL leads both PP and TG by a small but consistent margin.
3. All quants degrade smoothly with context depth; no instability observed.

## Practical takeaway

- **If interaction quality at deep context matters most:** IQ4_NL is the best default on this hardware.
- **If prefill-heavy workflows dominate at ~100k:** Q8/Q6 are competitive, but gap narrows and reverses near max context.

## Next step (required before final quant decision)

Run a **quality eval pass** (same prompt set on all 3 quants: reasoning, coding, extraction, instruction-following, multilingual) before locking production default.
